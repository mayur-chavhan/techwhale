Master the Linux Terminal: 50 Essential Commands You Need to Know

Mayur Chavhan December 26, 2022

As a Linux user, the terminal is an essential tool for interacting with your operating system. From installing and updating software to managing files and performing advanced tasks, the terminal allows you to accomplish a wide range of tasks more efficiently than using a graphical user interface (GUI).

Here are 50 important Linux commands you should know:

cd - Change the current directory ls - List the files in a directory pwd - Print the current working directory mkdir - Create a new directory rmdir - Remove an empty directory rm - Remove a file or directory cp - Copy a file or directory mv - Move a file or directory cat - Display the contents of a file less - View the contents of a file one page at a time head - Display the first few lines of a file tail - Display the last few lines of a file grep - Search for patterns in a file find - Search for files based on criteria sort - Sort the lines in a file uniq - Remove duplicate lines from a file wc - Count the number of lines, words, and characters in a file chmod - Change the permissions of a file or directory chown - Change the owner of a file or directory sudo - Run a command with superuser privileges su - Switch to a different user account whoami - Display the current user’s username passwd - Change the current user’s password ping - Test a network connection traceroute - Trace the route packets take to a destination telnet - Connect to a remote server using the Telnet protocol ssh - Connect to a remote server using the SSH protocol scp - Copy files between servers using the SSH protocol ftp - Transfer files using the FTP protocol rsync - Synchronize files between servers tar - Create and extract archives gzip - Compress and decompress files bzip2 - Compress and decompress files unzip - Extract files from a ZIP archive gunzip - Decompress a GZIP archive bunzip2 - Decompress a BZIP2 archive apt-get - Install and manage packages on Debian-based systems yum - Install and manage packages on Red Hat-based systems dnf - Install and manage packages on Fedora-based systems systemctl - Manage system services service - Manage system services journalctl - View the system log dmesg - View the kernel log top - Display information about the top CPU and
